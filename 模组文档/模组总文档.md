## 背景：2026-07-19 移除补货修复功能

本模组原名 "RSMF Restock Fix"，最初是主模组 RimSimManagementFramework（RSMF，边缘模拟经营框架）的 Harmony 附属模组，包含三块内容：①补货可靠性修复（整段替换主模组补货方法）；②商品管理器库存过滤 + 仓库库存列；③搬运机械体参与补货。

主模组 2026-07-18 更新已**原生修复补货逻辑**（补货周期机制 `activeRestockCycles`/`GetNeededForQueue`、阈值 0 = "售罄才补"、强制补货等）。本模组原有的补货修复补丁会压制主模组新实现，因此**全部删除**，模组随之改名转型，只保留与主模组无冲突的两项增强 + 可选 ASF 兼容。

## 保留功能一：商品管理器库存过滤 + 仓库库存列

### 玩法

打开商店货柜（Building_SimContainer）的"商品管理"窗口（Dialog_GoodsManager）后：

- 窗口顶部新增一行控制条：
  - **库存商品**（默认）：列表只显示"本图仓库实有"或"当前草稿已启用"的商品，避免在几百种已注册商品里翻找。
  - **全部商品**：恢复框架原行为，显示当前分类全部已注册商品。
  - **刷新库存**按钮：强制重建本图库存快照。
  - 右侧小字摘要："库存商品 X / 已注册 Y"。
- 商品列表右侧新增**仓库库存列**，逐行显示该商品在本图仓储中的总数量。
- 库存模式下的批量全选/清空（ToggleAllCurrentDef）被接管：只作用于"当前搜索结果的可见商品"，并自动按货柜剩余容量（扣除隐藏商品占用）做容量裁剪，超出部分自动下调目标数并弹消息提示（AutoTrimNotice）。

### 机制

- 库存统计由 `MapInventorySnapshotCache` 按 Map 缓存快照（120 tick 有效期，带 generation 计数），避免窗口每帧全图扫描。
- 统计口径两种（设置里切换）：
  - **严格模式**（默认）：只计有效玩家仓储位置（库存区/玩家货架等），排除货柜虚拟库存。
  - **宽松模式**：计所有玩家可用、未禁用的已生成物品，仍排除其他地图、未生成物品与货柜虚拟库存。
- 适配器链：`AdaptiveStorageInventorySourceAdapter`（ASF 模组建筑内物品，纯反射软兼容）→ `VanillaInventorySourceAdapter`（全图 Thing 扫描，按口径过滤），用 processedThingIds 去重。
- 补丁方式：Harmony Prefix/Postfix 挂 `Dialog_GoodsManager` 的 `DrawMainPanel`/`DrawHeader`/`DrawItemRow`/`GetFilteredItems`/`ToggleAllCurrentDef`，全部经反射探针定位（见 `GoodsManagerPatches.ValidateTargets()`），探针缺失则对应补丁 Prepare 返回 false 安全降级，启动日志列出缺失项。

## 保留功能二：搬运机械体参与补货

### 玩法

原版只会 Hauling 的殖民地机械体（如搬运机 Mech_Lifter）可担任商店"补货"岗位并真正执行补货。用法：经商管理 → 店员 → 店员配置 → 岗位列表 → 补货，把搬运机加入补货岗位。机械体处于充电/关机/护送等工作模式时不会抢补货活。

### 机制（补丁点）

主模组补货 WorkGiver 绑定自定义工种 Restocking，而搬运机原生工种只有 Hauling，被三层关卡挡住。本模组逐点打通（不改框架 DLL）：

1. `ShopStaffUtility.MechanicalStaffCanUseWorkType` Postfix：Hauling 机视为可干 Restocking。
2. `Pawn.WorkTypeIsDisabled` Postfix（特性方式补丁原版方法）：Hauling 机的 Restocking 不再禁用。
3. `MapComponent_RestockTaskQueue.CanUseRestockWorkGiver` Postfix：队列主动派工对 Hauling 机放行。
4. `MapComponent_MechShopStaffDispatcher.IsIdleForShopDispatch` Postfix：放宽机械体"空闲"判定（Wait/Wander/GotoWander/HaulToCell/Refuel 等也算空闲，充电不算）。
5. `MapComponent_MechShopStaffDispatcher.ShouldTryDispatch` Postfix：框架判定可派工时，若机械体工作模式非 Work 则压回 false。

全部为软降级设计：探针（`MechRestockPatchTargets`，4 个字符串反射私有方法）找不到 → Prepare 返回 false → 该补丁不启用，仅启动时 Warning 一次。

## 可选兼容：Adaptive Storage Framework

若安装了 ASF（packageId `adaptive.storage.framework`），仓库库存统计会把 ASF 建筑内"允许存入"的物品计入；通过 `AccessTools.TypeByName("AdaptiveStorage.ThingClass")` 纯反射接入，未安装 ASF 时静默跳过，运行期异常自动禁用并 Warning 一次。

## 设置项（选项 → 模组设置）

| 设置 | 默认值 | Scribe 键 | 说明 |
| --- | --- | --- | --- |
| 宽松库存识别 | 关 | `looseInventoryRecognition` | 切换库存统计口径；切换时全图快照缓存失效 |
| 搬运机械体参与补货 | 开 | `enableMechRestock` | 关闭后恢复框架原行为（机械体不能补货） |

已移除的旧补货修复设置（`enableRestockInvariantFix`、`enableThresholdSemanticFix`、`enableReservationIdempotencyFix`、`enableQueueSelfHealFix`、`enableRestockUiAssist`、`enableRestockWatchdog`、`verboseRestockLog`）不再读取；旧设置文件中的对应节点会被 Scribe 静默忽略，无报错。

## 构建

```powershell
dotnet build "Source/RSMFInventoryGoodsFilter/RSMFInventoryGoodsFilter.csproj" --configuration Release --nologo
```

输出 `Assemblies/RSMFInventoryGoodsFilter.dll`。本机主模组路径不在 csproj 默认位置，需模组根目录的 `Directory.Build.props`（已 gitignore，不提交）覆盖 `SimManagementLibDll`，模板见 `Directory.Build.props.example`。
