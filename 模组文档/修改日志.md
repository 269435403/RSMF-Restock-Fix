## 2026-07-19 — 移除补货修复功能，模组改名转型

### 为什么

主模组 RimSimManagementFramework 于 2026-07-18 原生修复补货逻辑（补货周期机制 `activeRestockCycles`/`GetNeededForQueue`、阈值 0 = "售罄才补"、强制补货等）。本模组原有的补货修复补丁（整段替换主模组方法）会压制主模组新实现，必须删除。模组保留两项无冲突功能并改名：`RSMF Restock Fix` → `RSMF 商品管理增强 (Goods Manager Plus)`。packageId `yyyyy.rsmf.restockfix`、本地文件夹名、程序集名、命名空间、Harmony ID 均不变（创意工坊已发布）。

### 删除了什么（12 个源文件）

- `Source/RSMFInventoryGoodsFilter/Restock/` 全部 7 个：`RestockFixGate.cs`、`RestockInvariant.cs`、`RestockPatchTargets.cs`、`RestockRepairService.cs`、`RestockReservationTracker.cs`、`RestockStatusPresenter.cs`、`RestockWatchdogMapComponent.cs`（watchdog 本就是从未注入地图的死代码）。
- `Source/RSMFInventoryGoodsFilter/Patches/` 中 5 个：`RestockApplySettingsPatches.cs`、`RestockNeedPatches.cs`、`RestockQueuePatches.cs`、`RestockReservationPatches.cs`、`RestockUiPatches.cs`。

### 怎么改的

1. **入口** `InventoryGoodsFilterMod.cs`：移除 `RestockPatchTargets.ValidateAndLog()` 调用与 `using ...Restock;`；改为 `GoodsManagerPatches.ValidateTargets()` → `MechRestockPatchTargets.ValidateAndLog()` → `PatchAll()`。
2. **机械体补丁** `MechRestockWorkTypePatches.cs`：
   - 4 个反射探针（`ShopStaffUtility.MechanicalStaffCanUseWorkType`、`MapComponent_RestockTaskQueue.CanUseRestockWorkGiver`、`MapComponent_MechShopStaffDispatcher.IsIdleForShopDispatch`、`ShouldTryDispatch`）原本就以 `TargetMethod()` 内联在本文件，现集中到新增的静态探针类 `MechRestockPatchTargets`，启动时统一 Warning；Prepare 找不到目标返回 false 静默降级的设计不变。
   - 门控由被删的 `RestockFixGate.IsMasterEnabled` 改为本地 `MechRestockEnabled`（读新设置 `EnableMechRestock`，默认开）；去掉对 `VerboseRestockLog` 的引用，启动提示固定一条 Message。
3. **设置** `InventoryGoodsFilterSettings.cs`：删除 7 个补货修复字段（`EnableRestockInvariantFix` 及 5 个子开关、`VerboseRestockLog`）与设置界面的"可靠自动补货"分区；保留 `LooseInventoryRecognition`（Scribe 键 `looseInventoryRecognition` 不变），新增 `EnableMechRestock`（新键 `enableMechRestock`，默认开）。
4. **商品管理器补丁** `GoodsManagerPatches.cs`：零改动——它本就自包含（自己的探针 + 主模组 public API `GoodsItemData.NormalizeRestockThreshold`），不引用任何被删类。
5. **翻译**：中英两文件删除 54 个仅补货修复使用的键；保留 20 个商品管理器/通用键；新增 4 个键（`Settings.MechSection`、`Settings.EnableMechRestock`、`Settings.EnableMechRestockTip`，以及补上原本代码在用但语言文件缺失的 `RSMF.GoodsManager.AutoTrimNotice`——既有 bug 顺手修复）；`SettingsCategory` 键值改为新模组名。
6. **About.xml**：`<name>` 改为 `RSMF 商品管理增强 (Goods Manager Plus)`；description 重写为只描述保留功能，注明补货修复已由框架原生实现。packageId/author/依赖/loadAfter 不变。
7. **README.md**：同步改名与功能描述，删补货修复章节，注明移除背景。
