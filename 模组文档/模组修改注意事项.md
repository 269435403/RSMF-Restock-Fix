# 模组修改注意事项

> 记录该模组特有的坑；单看总文档容易遗漏的细节。

## 1. 主模组 DLL 编译期直接引用，类型/方法改名会炸

`RSMFInventoryGoodsFilter.csproj` 直接引用 `SimManagementLib.dll`（主模组 RimSimManagementFramework 的程序集）。在 C# 源码中直接 `typeof()` 了若干主模组类型（如 `SimManagementLib.SimDialog.Dialog_GoodsManager`、`SimManagementLib.SimThingClass.Building_SimContainer` 等）。

**后果**：如果主模组在更新中**改了类型名、方法名或方法签名**，编译时直接报 `TypeLoadException`/`MissingMethodException`，无法通过错误。不同于纯反射的软探测，这是硬引用。

**应对**：
- 主模组更新后必须先编译，编译错误就找到对应引用点改成新版签名。
- 尽量使用主模组仍然保留的 public API；当前被保留的如 `GoodsItemData.NormalizeRestockThreshold`（在 `ClampVisibleCapacity` 中调用）如果哪天改了签名必须同步改。

## 2. csproj 的 HintPath 是机器路径，需本地 `Directory.Build.props` 覆盖

`csproj` 默认的 `SimManagementLibDll` 路径是：
```xml
$(RimWorldDir)\Mods\RimSimManagementFramework\Assemblies\SimManagementLib.dll
```

本机主模组实际在：
```
Mods\模组制作工作台\边缘模拟经营框架RSMF相关\RimSimManagementFramework边缘模拟经营框架\Assemblies\SimManagementLib.dll
```

**必须**在模组根目录放 `Directory.Build.props` 覆盖 `SimManagementLibDll`；该文件已列入 `.gitignore`，**不要提交**。`Directory.Build.props.example` 已在仓库中提供模板。

## 3. 机械体补丁的探针全是反射私有方法，随主模组改名即失效

`MechRestockWorkTypePatches.cs` 用 `AccessTools.Method` 探测 4 个主模组私有方法：
- `ShopStaffUtility.MechanicalStaffCanUseWorkType`
- `MapComponent_RestockTaskQueue.CanUseRestockWorkGiver`
- `MapComponent_MechShopStaffDispatcher.IsIdleForShopDispatch`
- `MapComponent_MechShopStaffDispatcher.ShouldTryDispatch`

**这些名字在代码中是以字符串硬编码**，所以主模组改名后不会编译错误，但补丁会静默失效。启动日志里会输出 `Warning`（探针找不到），机械体补货功能退化到框架原行为。修改时如果主模组改了这些方法名，必须同步改字符串。

## 4. 设置字段的 Scribe 键名不能随意改（玩家设置持久化）

`InventoryGoodsFilterSettings.cs` 中 `Scribe_Values.Look(..., key, ...)` 的 key 一旦写入玩家存档，就不要改，否则旧存档里的设置值会丢失并回退到默认值。

本次改造后的存活键：
- `looseInventoryRecognition`（旧键，保留）
- `enableMechRestock`（新键，默认 true）

已废弃的旧键（`enableRestockInvariantFix`、`enableThresholdSemanticFix` 等）不再读取，但玩家旧存档中仍会有这些节点，Scribe 会静默忽略，不报错。

## 5. packageId 已发布，不可改

`About/About.xml` 中的 `packageId` 为 `yyyyy.rsmf.restockfix`。该模组已在 Steam 创意工坊发布，修改 `packageId` 会导致已订阅用户出现模组重复或依赖断裂。因此即使模组改名，packageId 必须保持原值。

## 6. `MechRestockWorkTypePatches` 中 `Pawn.WorkTypeIsDisabled` 是特性方式补丁，要确保 `HarmonyPatch` 目标在 RimWorld 1.6 中仍可用

`Pawn_WorkTypeIsDisabled_Patch` 使用的是 `[HarmonyPatch(typeof(Pawn), nameof(Pawn.WorkTypeIsDisabled))]`，如果 RimWorld 未来版本（尽管我们当前只支持 1.6）改了该方法签名或重命名，特性匹配会编译失败。

## 7. 不要动 `About/Preview.png` 和 `Preview/` 文件夹

用户自行处理，模组代码/构建不涉及这些文件。git 不要提交预览图修改。

## 8. `RSMF.GoodsManager.AutoTrimNotice` 键在 2026-07-19 之前缺失，已补齐

`GoodsManagerPatches.ToggleVisibleItems()` 在代码中使用了 `RSMF.GoodsManager.AutoTrimNotice.Translate(...)`，但之前的语言文件里没有这个键。2026-07-19 改造时顺手补上中英两语言条目。
